## 你是谁，你在做什么

你正在协助开发 **Personal OS** —— 一套个人长期管理系统。这不是一个普通项目，它要**运行 10 年以上**。你写的每一行代码、每一份文档都要经得起时间检验。

## 核心原则（务必遵守）

1. **保持简单** — 不要为想象中的需求设计。当前用不到的抽象，就别建。
2. **长期维护** — 选择无聊、稳定、可读的技术。SQLite 优于花哨的数据库。
3. **不过度设计** — 一个文件能解决的事不要拆成五个；一个表能解决不要分库。
4. **数据优先** — 数据是核心资产。永远不要让数据依赖某个会消失的服务。
5. **AI 只作为分析工具** — AI 不持有数据、不替代数据存储、不接管决策。

## 开发约定

### 代码风格
- Python：使用类型注解、简洁函数、清晰命名。
- API 路由统一前缀 `/api`。
- 时间统一用 ISO 8601 (UTC) 存储，展示时转本地。

### 数据库
- 新增表前先读 [DATABASE.md](./DATABASE.md)。
- schema 变更走迁移脚本，保留向后兼容。
- 默认主键用自增整数，必要时用 ISO 时间戳排序。

### 版本
- 遵循语义化版本 (SemVer)。
- 每次有意义的变更更新 [CHANGELOG.md](./CHANGELOG.md)。

### 任务
- 新工作登记到 [TASKS.md](./TASKS.md)，编号 `Task-XXX`。

## 工作方式

- **先读再写**：动手前先读相关文档和代码。
- **小步前进**：一个任务一个提交，可独立验证。
- **诚实汇报**：完成了就说完成，跳过了就说跳过，失败了就说失败。
- **不擅自提交 Git**：除非用户明确要求，不要执行 `git commit` 或 `git push`。用户会自行处理 Git。
